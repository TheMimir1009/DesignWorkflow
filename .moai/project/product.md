## 프로젝트 소개

### 제품 비전

**DesignWorkflow**는 게임 기획 문서 생성을 위한 AI 기반 워크플로우 시스템입니다. 게임 기획자가 아이디어를 입력하면, AI가 체계적인 질문-응답 과정을 거쳐 전문적인 기획 문서와 프로토타입 코드를 자동으로 생성합니다.

### 핵심 가치

- **효율성**: 수작업 문서 작성 시간을 대폭 단축
- **일관성**: 표준화된 문서 형식과 품질 보장
- **확장성**: 다중 LLM 지원으로 유연한 AI 활용
- **추적성**: 칸반 보드를 통한 작업 진행 상황 시각화

## 대상 사용자

### 주요 사용자 유형

| 사용자 유형 | 역할 | 주요 사용 기능 |
|------------|------|--------------|
| **게임 기획자** | 피처 설계 및 문서 작성 | 피처 등록, Q&A 응답, 생성 문서 검토/수정 |
| **프로젝트 매니저** | 프로젝트 관리 및 모니터링 | 대시보드 확인, 프로젝트 설정, 진행 상황 추적 |
| **개발자** | 프로토타입 활용 | 생성된 Prototype 코드 참조 및 구현 |

### 사용자 페르소나

**게임 기획자 (주 사용자)**
- 게임 메카닉, 경제 시스템, 성장 시스템 설계를 담당
- 빠른 아이디어 구체화와 문서화 필요
- 기술적 배경 지식은 다양한 수준

**프로젝트 매니저**
- 여러 피처의 진행 상황 파악 필요
- 팀 전체의 작업 현황 모니터링
- 리소스 배분과 일정 관리 담당

## 핵심 기능

### 1. 칸반 보드 (4단계 워크플로우)

게임 기획 문서 생성의 전체 과정을 시각화하는 4단계 파이프라인입니다.

```
Feature List → Design Doc → PRD → Prototype
```

| 단계 | 설명 | 주체 |
|------|------|------|
| **Feature List** | 사용자가 기능 아이디어 입력 | 사용자 |
| **Design Doc** | Q&A 기반 상세 설계 문서 생성 | Q&A + AI |
| **PRD** | 제품 요구사항 문서 자동 생성 | AI |
| **Prototype** | 프로토타입 코드 자동 생성 | AI |

### 2. Q&A 시스템

Design 단계 진입 시 카테고리별 구조화된 질문을 통해 요구사항을 수집합니다.

**지원 카테고리**:
- **Game Mechanic**: 게임 메카닉 관련 질문
- **Economy**: 게임 경제 시스템 관련 질문
- **Growth**: 캐릭터/플레이어 성장 시스템 관련 질문

### 3. AI 문서 생성

LLM을 활용하여 수집된 정보를 바탕으로 전문적인 문서를 자동 생성합니다.

**생성 문서 유형**:
- Design Document (설계 문서)
- PRD (제품 요구사항 문서)
- Prototype Code (프로토타입 코드)

### 4. 다중 LLM 지원

다양한 AI 제공자를 유연하게 활용할 수 있는 추상화 레이어를 제공합니다.

| 제공자 | 특징 |
|--------|------|
| **OpenAI** | GPT-4o, GPT-4o-mini 등 최신 모델 지원 |
| **Google Gemini** | Gemini Pro 모델 지원 |
| **Claude** | Claude Code 통합 |
| **LM Studio** | 로컬 LLM 서버 연동 (오프라인 사용 가능) |

### 5. 참조 문서 관리

AI 컨텍스트 강화를 위한 시스템 문서 관리 기능을 제공합니다.

- 프로젝트별 참조 문서 등록
- 태스크별 참조 문서 선택
- AI 생성 시 컨텍스트로 자동 포함

### 6. 프롬프트 관리

문서 생성용 프롬프트 템플릿을 커스터마이징할 수 있습니다.

- 단계별 프롬프트 템플릿 설정
- 변수 치환 지원 (프로젝트명, 기술 스택 등)
- 프로젝트별 프롬프트 커스터마이징

### 7. 대시보드

프로젝트 현황을 한눈에 파악할 수 있는 모니터링 기능을 제공합니다.

- 단계별 태스크 분포 차트
- 최근 활동 내역
- 프로젝트 통계 요약

## 사용 사례

### 사용 사례 1: 새로운 게임 피처 기획

**시나리오**: 게임 기획자가 새로운 "가챠 시스템"을 기획하려고 합니다.

**워크플로우**:
1. Feature List에 "가챠 시스템" 피처 생성
2. Design 단계로 이동 시 Q&A 모달 표시
3. Economy 카테고리 질문에 응답 (확률, 보상, 천장 시스템 등)
4. AI가 Design Document 생성
5. PRD 단계로 이동하여 상세 요구사항 문서 생성
6. Prototype 단계에서 기본 구현 코드 생성

### 사용 사례 2: 기존 시스템 문서화

**시나리오**: 이미 구현된 전투 시스템의 문서가 필요합니다.

**워크플로우**:
1. 시스템 문서 사이드바에 기존 전투 시스템 코드/문서 등록
2. 새 피처 생성 시 참조 문서로 선택
3. AI가 기존 시스템을 참조하여 일관된 문서 생성

### 사용 사례 3: 팀 협업 및 진행 상황 추적

**시나리오**: PM이 여러 기획자의 작업 현황을 파악해야 합니다.

**워크플로우**:
1. 대시보드에서 전체 프로젝트 현황 확인
2. 칸반 보드에서 각 피처의 진행 단계 확인
3. 지연된 작업 식별 및 리소스 재배분
